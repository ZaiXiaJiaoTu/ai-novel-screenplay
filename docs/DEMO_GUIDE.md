# 演示指南

本文档用于评审或本地验收时快速复现“小说导入 -> 大模型处理 -> 剧本生成 -> 剧本导出”的完整流程。

## 前置条件

- PostgreSQL 已启动，并已执行 `sql/init.sql`。
- 后端已启动：`http://127.0.0.1:8000`。
- 前端已启动：`http://127.0.0.1:5173`。
- 如需真实大模型生成效果，准备一个兼容 OpenAI 格式的 API Key，例如 DeepSeek。

## 1. 配置大模型

进入“模型配置”页面，点击“新增配置”：

- 供应商：`deepseek`
- Base URL：`https://api.deepseek.com/v1`
- 模型名称：`deepseek-chat`
- API Key：填写你的 API Key
- Temperature：`0.7`
- Top P：`0.9`
- Max Tokens：`4096`
- 勾选“启用配置”和“设为默认”

保存后点击“测试”。测试通过后，后续生成流程会优先使用该配置。

如果没有 API Key，系统仍可通过占位/降级结果打通演示链路，但生成质量会明显低于真实模型输出。

## 2. 初始化提示词模板

进入“提示词模板”页面，点击“初始化默认模板”。

默认模板覆盖以下任务：

- `story_profile_generation`
- `chapter_summary_generation`
- `style_strategy_generation`
- `scene_plan_generation`
- `script_yaml_generation`
- `yaml_repair`

如果重复点击，后端会跳过已存在的同类型模板。

## 3. 可选：初始化演示数据

如果只想快速查看完整演示效果，可以在项目根目录执行：

```powershell
cd backend
.\.venv\Scripts\python.exe -m app.scripts.seed_demo_data
```

该命令会创建演示作品《夜雨旧楼》、章节摘要、故事设定档案和一个已生成的短剧项目。重复执行不会创建重复作品。

如果需要演示从零导入到生成的完整流程，可以跳过本步骤，继续手动导入小说。

## 4. 导入小说

进入“待选书架”页面，可选择两种方式：

- 粘贴文本：填写作品名称和小说正文。
- 上传文件：上传 `.txt` 文件，可选填写作品名称。

建议正文包含明确章节标题，例如：

```text
第一章 旧楼来信
林川在雨夜回到旧楼，发现门缝里夹着一封没有署名的信。

第二章 消失的房间
他按信上的地址上楼，却发现那间房在十年前已经被封死。
```

导入后页面会显示作品列表和章节列表。

## 5. 生成章节摘要

在“待选书架”页面选择作品，点击“生成章节摘要”。

结果会写入章节摘要表，并可在章节列表中点击单章查看摘要。

## 6. 生成剧本

进入“剧本书架”页面，点击“生成剧本”：

- 选择刚导入的作品。
- 设置剧本类型、风格、压缩级别、章节范围和目标时长。
- 点击“创建并启动”。

当前任务为同步执行。短文本通常会很快完成；长文本或真实模型调用可能需要等待。

任务完成后，剧本项目会出现在“项目列表”，片段会出现在“片段列表”。

## 7. 编辑和导出

在“剧本书架”页面：

- 点击片段行或“编辑”查看 YAML 和纯文本内容。
- 修改内容后点击“保存”。
- 点击片段或项目上的 `YAML` / `TXT` 下载入口导出。

## 8. 查看调用日志

进入“调用日志”页面：

- 可按任务类型、状态、时间范围筛选。
- 点击日志行查看请求摘要、响应摘要、错误信息、Token 用量和耗时。

如果生成失败，优先检查这里的错误摘要，再检查“模型配置”和“提示词模板”。

## 常见问题

### 数据库连接失败

确认 `backend/.env` 中的 `DATABASE_URL` 与本地 PostgreSQL 一致。默认值为：

```env
DATABASE_URL=postgresql+psycopg://postgres:postgres@localhost:5432/ai_novel_screenplay
```

### 前端接口请求失败

确认后端在 `http://127.0.0.1:8000` 运行，前端 Vite 代理会把 `/api` 转发到该地址。

### 没有真实大模型输出

确认：

- 模型配置已启用。
- 至少一个配置被设为默认。
- Base URL、模型名称和 API Key 正确。
- 提示词模板已初始化且处于启用状态。
